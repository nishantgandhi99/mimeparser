New features overview

The new features in MIMEsweeper for SMTP relate to the following areas:

  • Edge Integration

    In a MIMEsweeper for SMTP installation, you can now incorporate Edge server appliances to handle specific scenarios such as virus detection and blocking, and spam detection meaning that the MIMEsweeper environment does not need to perform all of the spam filtering. You can also monitor these servers using MIMEsweeper Manager. The Edge servers will also appear in the System Health and Message Center areas of MIMEsweeper Manager.

    See the MIMEsweeper Manager Online Help for information on configuring and using Edge servers in your deployment.

  • PMM Enhancements

    In an environment that uses Windows Authentication, users can now configure their PMM setups to manage spam messages sent to an organization’s public folders. For example, a designated user could manage the spam messages sent to contact addresses advertised on an organization’s website.

    See the PMM Online Help for information.

  • Database and authentication changes

    • The default database server delivered with the installation software for new customers uses SQL Server 2005 Express which provides a larger maximum database size, and additional management functionality. The database server for existing customers is unchanged from MSDE.

    • In addition to SQL Server authentication supported in previous versions, this release supports Windows Authentication for MIMEsweeper for SMTP's database operations. If upgrading, Windows Authentication can be configured after installation using the System Maintenance application. For a new installation, Windows Authentication is available for selection in the installation procedure.

    In the Getting Started Guide, in Chapter 3, Installation, see the Preparing the Database Environment section for information on setting up the database software, and configuring your MIMEsweeper software to use Windows authentication.

  • Lexical Analysis Enhancements

    MIMEsweeper for SMTP lexical analysis now supports PERL/POSIX syntax for defining regular expressions to detect text. The REGEXP syntax available in previous versions is also supported. In addition, lexical analysis can now detect Credit Card Numbers, US Social Security Numbers, and UK National Insurance Numbers. Performance has also been improved.

    In the MIMEsweeper for SMTP Reference, in Chapter 5, References, see Expression List Properties and Configuring Regular (PERL/POSIX) Expressions for more information.

  • Document Analysis Enhancements

    You can now detect text in:

    • Headers, footers and document properties in a Microsoft Office document.
    • Headers, footers and document properties in an OpenOffice document.
    • Document properties in a PDF document.

    We now support decomposition of Office 2007 and OpenOffice file formats.

    In the MIMEsweeper for SMTP Reference, in Chapter 4, Scenarios, see Text Analyzer for more information.

  • Multi-Domain Management

    You can now configure MIMEsweeper for SMTP Mail administrators to restrict the mail domains within the organization that they can manage. Administrators can manage only those messages originating from, or addressed to, the mail domains for which they have administration rights. The MIMEsweeper 5.3 system can support in excess of 1000 domains.

    See the MIMEsweeper Manager Security Center online help for details.

  • Additional formats

    The Data Types scenario now has the ability to detect the following additional formats:

    • 3GP Multimedia.

    • Advanced System Format (ASF).

    • Flash video (FV).

    • Free Lossless Audio Code (FLAC).

    • Google video (GV).

    • Monkey’s Audio (APE).

    • MP4 Audio.

    • Ogg Vorbis compressed audio file.

    • PlayStation Portable Multimedia.

    • Sun Sound File (AU).

    • HTML encoded within RTF (HTML).

    • HTML with script (HTML).

    • Microsoft Office Embedded Objects.

    • Microsoft Office editdata.mso.

    • Microsoft Outlook file attachment.

    • Open Office Calc Document (ODS).

    • Open Office Chart.

    • Open Office Graphic Document (ODG).

    • Open Office Impress Document (ODP).

    • Open Office Master Document (ODM).

    • Open Office Math Document (ODF).

    • Open Office Writer Document (ODT).

    • Open Package Convention.

    • Pattern Matched.

    • StarView Metafile (SVM).

    • ALSI Protected Documents.

    • NEC InfoCage protected format.

    • Pointsec encrypted files.

    • Microsoft Visio Document (VSD).

    • Initial Graphics Exchange Specification (IGES).

    In the MIMEsweeper for SMTP Reference, see Appendix B, Data Types for more information.

  • Pattern Matcher Enhancements

    The Pattern Matcher scenario now has the additional ability to search a file, in reverse, from the end of the file to the start with an optional offset back from the end of file marker. The Text Analyzer has also changed to reflect changes to the scanning criteria.

    In the MIMEsweeper for SMTP Reference, in Chapter 4: Scenarios, see Pattern Matcher for more information.

Standby MIMEsweeper server (Available in Advanced and Enterprise versions only)

  • For deployments that require high availability, it is recommended that you set up a standby server when planning your MIMEsweeper deployment, to provide a contingency machine in the event of an emergency or increased message processing activity. This additional server can be installed with all or any of the MIMEsweeper deployment components, and can be promoted to Primary Configuration Server (PCS) status assuming the MIMEsweeper machine meets the minimum requirements as specified in the Prerequisites for MIMEsweeper for SMTP 5.3 release document.

    • To set up a standby server in your MIMEsweeper deployment:

    To set up a standby server, you must follow the installation Route 4 as specified in the MIMEsweeper for SMTP Getting Started Guide.

    This installation route describes the process for configuring a custom install type then installing an additional server. The MIMEsweeper machine can be set up with all the MIMEsweeper deployment components but, if preferred, the machine can be disabled.

  • Although the Clearswift products will install and function adequately within virtualization software, the performance characteristics of virtual machine hardware can be significantly different from dedicated hardware solutions. The potential adverse effects on the operational running of Clearswift products on virtual hardware solutions have currently not been fully quantified.

Web server naming issues

  • In the MIMEsweeper for SMTP installation, web servers that host web applications, that is MIMEsweeper Manager, and Personal Message Manager (PMM) should not have any underscore characters (_) in their names. An underscore in a web server's name can affect the way that Internet Explorer maintains session state and cookie information.

Invalid Expressions in the Expression List references

  • When it starts, the MIMEsweeper for SMTP Security Service now checks for invalid expressions defined in any Expression List references.

    In MIMEsweeper 5.3 the Policy Editor now prevents you from entering an invalid expression. In earlier versions, invalid expressions were allowed. If you use Expression Lists in your policy, when you complete an upgrade from the previous version, check the MIMEsweeper Log for evidence of invalid expressions.

    To check the log:

    1. From MIMEsweeper Manager, access the System Center, and click on a server link to open the server's page.
    2. On the server's page, click on the Logs tab, and from the Log Name: drop-down list, select MIMEsweeper log to display the log entries.
    3. Look for entries of the form:
      There is a problem with the Expression name user-defined expression list - invalid expressions will be ignored.
      This is followed by an error message indicating the expression is invalid. A further error message describes the problem.
      These errors do not prevent the Security Service from starting, and the invalid expressions are ignored.

Using the System Maintenance Utility to move the Operations Database

  • If you attempt to use the System Maintenance Utility to move the Operations Database, ensure that the database software on the target machine is the same version or higher than the database software on the source machine. If the target machine database software version is lower than the source machine software version, the move process fails.

Quarantine Messages Batch Delete operation can sometimes fail to complete

  • With some browser configurations, the MIMEsweeper Manager Quarantine Messages Batch Delete operation can fail to complete, and the Batch Operation Status screen fails to close. If this problem occurs, configure Internet Explorer as follows:

    1. From the Tools menu, select Internet Options.
    2. On the Internet Options page, select the Security tab, and then click the Custom level button.
    3. In the Settings list, scroll down to the Miscellaneous, Allow META REFRESH item.
    4. Select the Enable radio button for the item, and then click OK, and OK again to save the settings.

Back to Problems known in this release

Manager and System Health displays can sometimes fail to initialize, or can fail to refresh after initialization

  • As with the MIMEsweeper Manager Quarantine Messages Batch Delete operation issue, this is caused by the Internet Explorer browser configurations. If this problem occurs, configure Internet Explorer as follows:

    1. From the Tools menu, select Internet Options.
    2. On the Internet Options page, select the Security tab, and then for the appropriate Security zone click the Custom level button.
    3. In the Settings list, scroll down to the Miscellaneous, Allow META REFRESH item, and select the Enable radio button for the item.
    4. Scroll further down the list to the Scripting, Active scripting item and select the Enable radio button for the item.
    5. Click OK, and OK again to save the settings.

MIMEsweeper Manager will not initialize when using Windows 2008

  • MIMEsweeper Manager can fail to initialize when using Windows 2008 due to the Enhanced Security Configuration option being enabled. If this problem occurs, configure the Internet Explorer Enhanced Security Configuration (ESC) as follows:

    1. On the Internet Explorer Enhanced Security Configuration page, select Off for both Administrators and Users.
    2. Click OK to save the settings.

Windows Server 2008 Uninstall authorization problem

  • If you attempt to uninstall MIMEsweeper for SMTP running on a Windows Server 2008 platform using the Add/Remove Programs option, you may receive an Insufficient Access Permissions error. This is caused by the manner in which Windows Server 2008 handles access permissions.

    • To avoid this problem, you need to retain a copy of the MIMEsweeper for SMTP software that you used to install the software. To uninstall the software:

    1. Run the installation software's setup.exe program.
    2. From the Program Maintenance screen, select the Remove option, and follow the prompts to remove the application.

Authorization problems when running the System Maintenance utility on Windows Server 2008 machines

  • On Windows Server 2008 systems that have the User Account Control (UAC) feature enabled, you must run the System Maintenance Utility with Administrator privileges.

    To run the utility with administrator privileges, right-click the utility's shortcut, and from the menu, select Run as Administrator.

    To configure the shortcut to the System Maintenance Utility application so that you can run it as normal from the Start menu:

    1. Locate the shortcut to the program: In the Start, Programs menu, locate and right-click the System Maintenance Utility item, and select Properties. The shortcut's location is available under the General tab.
    2. Configure administrator privileges for the shortcut:
      1. Right-click the System Maintenance Utility shortcut, and from the drop-down list, select Properties.
      2. On the Properties dialog box, select the Compatibility tab, and select the Run this program as an administrator checkbox.
      3. Click OK to save the configuration.

Using Personal Message Manager (PMM) in NTLM environments

  • In environments that use NTLM, with some configuration combinations, users cannot access their PMM accounts. This can occur if:

    • MIMEsweeper for SMTP uses Windows Authentication.
    • PMM is set up to use Integrated Windows Authentication.
    • The MIMEsweeper for SMTP Web Server is logged on using a local administrator account.

    In these environments, the user account under which MIMEsweeper for SMTP Web Server operates must be logged in as a network administrator.

New PMM accounts with Windows Integrated PMM

  • In multi-domain environments that use Windows Integrated PMM, new account creation can fail if the mail domain part of a user's email address is not defined in the Policy Editor's SMTP Relay Domain settings. For users with multiple email addresses, the system is unable to create a PMM account for both the primary address, and any secondary mail addresses for these users.

    If a new PMM user is unable to login to PMM, check that the user's configuration meets the above requirements. You can check a user's primary and secondary addresses using the Active Directory Users and Computers application that runs your Primary Directory Server. To determine your Primary Directory Server:

    1. In MIMEsweeper Manager, access the Message Center and click the Configure PMM button.
    2. On the Configure Personal Message Manager page, select the Authentication Settings tab to display the directory server information.

Unable to create Audit database when using a named instance of SQL Server and Dynamic Ports

  • When the MIMEsweeper for 5.3 installer is used to install SQLExpress under an Instance Name (not the default option), SQLExpress is configured to use Dynamic Ports. Dynamic Port allocation means that SQL Server will select an available port when the service is started that may change if the service is subsequently stopped and restarted.

    In order for MIMEsweeper to access its SQL databases when running with Dynamic Ports, the SQL Server Browser Service must be enabled and started. This service maps requests from applications for a specific SQL instance to the port number that is currently assigned to that instance.

    When configuring MIMEsweeper Auditing and Message Tracking to use a Named Instance of SQLExpress, the auditing and tracking wizards request an Instance Name AND Port to define the connection. If Dynamic Ports are in use, this causes problems because the current port used by SQL may change at a later date.

    To use a Named Instance with Dynamic Ports:

    • Select the Database Connection tab of the Auditing and Message Tracking wizards.
    • In the Specify Server edit box, enter the Server Name followed by a backslash followed by the Instance Name, i.e. Server\Instance, for example, DBServer\MIMEsweeper.

    Configuring SQLExpress to use fixed ports

    To configure SQLExpress to use a fixed port instead of dynamic ports you should:

    1. Run the SQL Server Configuration Manager.
    2. Under the SQL Server 2005 network Configuration node, click Protocols for InstanceName (where InstanceName is the instance of SQLExpress).
    3. Double-click TCP/IP to display the Properties dialog.
    4. Click the IP Addresses tab.
    5. In the IP All section, change the TCP Dynamic Ports number to zero.
    6. Type the required port number in TCP Port, for example, 1433.
    7. Click Apply and then OK.
    8. The service running this instance of SQL will need to be restarted for the changes to take effect. The service will be called SQL Server (InstanceName), for example, SQL Server (SQLEXPRESS).

Task failure when using System Maintenance Utility tasks

  • An error message may be displayed when using the following System Maintenance Utility tasks:

    • Make this machine the Primary Configuration Server (Advanced License only).
    • Move the Operations Database to another SQL Server (Advanced License only).
    • Make this machine the Primary Configuration Server and Operations Database host (Standard License only).
    • Change Operations Database to use Windows Authentication.
    • Configure Web and Audit Applications to use Windows Authentication.

    The displayed error message will be as follows:


    Failed to grant the 'Log on as a service' permission to user 'DOMAIN\User'. Please use the Local Security Policy console to apply this manually before retrying this task.

    To rectify this problem, you need to grant the Log on as a service security right to the Application Service Account you entered during the task wizard, as follows:

    1. Open the Control Panel, select Administrative Tools then Local Security Policy management console.
    2. Drill down to User Rights Assignment within Local Policies.
    3. Open the Log on as a service properties page in the right-hand pane.
    4. Click Add User or Group and enter your Application Service Account.
    5. Click Apply to save the changes.

    After making this change, you should retry the failed task.

Installing pre-requisite software in a Windows Server 2008 environment

    Activating pre-requisite software

      • Windows Authentication within Security within Roles Services within Web Server (IIS) in the Server Manager.
      • IIS 6 Metabase Compatibility within IIS 6 Management Compatibility within Management Tools within Roles Services within Web Server (IIS) in the Server Manager.

    • For further information, refer to the Windows Server 2008 system documentation.

Text processing results view

  • The help topic that describes the Text Analysis results screen does not describe the c: and the w: tags.

    • The c: tag represents the position of the matching text as a character offset from the start of text, starting from zero.
    • The w: tag represents the offset position of the word that contains the matching text, from the start of the text.
